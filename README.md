# 🗺️ ExploreWithMe — Платформа-афиша с рекомендательной системой

Проект организован как многомодульный Maven-репозиторий, разделенный на логические слои:

### 1. 📂 Core (Бизнес-логика) — `/core`
Набор независимых микросервисов, реализующих основной функционал платформы:
* **`event`** — ядро системы: публикация, поиск и фильтрация мероприятий.
* **`user`** — управление профилями пользователей и правами доступа.
* **`request`** — обработка заявок на участие в событиях.
* **`comments`** — система отзывов и модерации контента.
* **`compilation`** — формирование подборок событий администраторами.
* **`common`** — общие DTO и утилитарные классы.

### 2. 📂 Recommendations (Аналитический контур) — `/recommendations`
Система умных рекомендаций, построенная на событийной модели (Event-Driven):
* **`collector`** — gRPC-сервер, принимающий потоки действий пользователей (просмотры, лайки, регистрации) и транслирующий их в Kafka.
* **`aggregator`** — расчет косинусного сходства между мероприятиями в поточном режиме.
* **`analyzer`** — движок предсказаний: хранит историю взаимодействий и выдает персональные рекомендации через gRPC API.
* **`recommendations-api`** — общие интерфейсы и Protobuf-схемы для рекомендательного контура.

### 3. 📂 Infra (Инфраструктура) — `/infra`
Облачная обвязка на базе **Spring Cloud**:
* **`config-server`** — централизованное хранение настроек.
* **`discovery-server`** (Eureka) — регистрация и обнаружение сервисов.
* **`gateway-server`** — единая точка входа с динамической маршрутизацией.

---

## 🛠 Технологический стек
* **Backend:** Java 11+, Spring Boot, Spring Cloud.
* **Communication:** **gRPC** (Protobuf) для внутренних вызовов, **REST** (Feign) для межсервисного взаимодействия.
* **Data Stream:** **Apache Kafka** (Avro) для передачи телеметрии действий пользователей.
* **Databases:** PostgreSQL (паттерн **Database per Service**), Hibernate / Spring Data JPA.
* **DevOps:** Docker, Docker Compose, Postman (коллекции тестов в папке `/postman`).

---

## 🚀 Ключевые фичи и решения
1. **Item-Based Collaborative Filtering:** Реализован алгоритм рекомендаций, который находит похожие события на основе весов действий пользователей (от простого просмотра до регистрации).
2. **Инкрементальный расчет дельт:** Для оптимизации нагрузки `aggregator` пересчитывает коэффициенты сходства только при поступлении новых данных, не перегружая систему полным пересчетом базы.
3. **Облачная готовность:** Благодаря `Gateway` и `Eureka` система легко масштабируется — можно запустить несколько экземпляров любого сервиса без изменения конфигурации.
4. **Типизация данных:** Использование Avro в Kafka и Protobuf в gRPC гарантирует строгое соблюдение контрактов между сервисами и высокую скорость сериализации.
